# Obsidian 설정 가이드

이 프로젝트를 Obsidian에서 최적으로 사용하기 위한 설정 가이드

## 1. Vault 열기

1. Obsidian 실행
2. "Open folder as vault" 선택
3. `/Users/jojongho/Projects/marketing` 폴더 선택

## 2. 설정 완료 상태

### ✅ 이미 설정된 항목
기존 Flow System 볼트의 설정이 복사되었습니다:
- **app.json**: 에디터 설정, 파일 경로 규칙
- **appearance.json**: Blue Topaz 테마, moonstone 다크 모드
- **community-plugins.json**: 선별된 핵심 플러그인 30개

### Core Plugins (기본 플러그인)
- ✅ Templates (템플릿 폴더: `templates/`)
- ✅ Daily notes
- ✅ Tags pane
- ✅ File explorer
- ✅ Search
- ✅ Quick switcher
- ✅ Backlinks

### Community Plugins (설치 완료)

#### 🔧 핵심 생산성 도구
- **Dataview**: 데이터 쿼리 및 대시보드 (필수)
- **Templater**: 고급 템플릿 기능 (필수)
- **QuickAdd**: 빠른 노트 생성 매크로
- **Omnisearch**: 강력한 전체 검색
- **Recent Files**: 최근 파일 빠른 접근
- **Various Complements**: 자동완성
- **Outliner**: 아웃라인 편집 강화
- **Note Refactor**: 노트 리팩토링 및 추출
- **Workspaces Plus**: 워크스페이스 관리

#### 📝 콘텐츠 작성
- **Editing Toolbar**: 편집 도구바
- **Admonition**: 콜아웃 블록 (노트, 경고 등)
- **Highlightr**: 형광펜 하이라이트
- **URL into Selection**: 링크 빠른 삽입
- **Obsidian Linter**: 마크다운 포맷 자동 정리

#### 🎨 시각화 & 관리
- **Excalidraw**: 다이어그램 그리기
- **Kanban**: 프로젝트 보드
- **Projects**: 프로젝트 관리 (테이블/보드/캘린더 뷰)
- **Tasks**: 작업 관리 및 추적
- **Calendar**: 캘린더 뷰
- **Icon Folder**: 폴더 아이콘
- **Folder Notes**: 폴더별 README 노트
- **Make.md**: 고급 노트 편집 및 관리

#### 🏷️ 조직화
- **Tag Wrangler**: 태그 관리 도구
- **Style Settings**: 테마 커스터마이징

#### 🤖 AI 지원
- **Copilot**: AI 채팅 도우미
- **Smart Composer**: AI 작문 지원
- **Text Generator**: AI 텍스트 생성
- **Smart Connections**: AI 기반 노트 연결 추천

### ⚠️ 제외된 플러그인
원래 65개 플러그인 중 복잡도 높거나 중복되는 플러그인 제외:
- Day Planner, Full Calendar, Week Planner (일정 관리 중복)
- Google Calendar, iCal (외부 연동 복잡도)
- Terminal, Execute Code (개발 전용)
- Pandoc, Mind Map, Graph Analysis (특수 용도)
- Book Search, YouTube Transcript 등 특수 기능
- 기타 35+ 플러그인

## 3. 볼트 활성화 절차

### 1단계: Obsidian에서 볼트 열기
```
Obsidian → Open folder as vault → /Users/jojongho/Projects/marketing 선택
```

### 2단계: Community Plugins 활성화
```
Settings → Community plugins → Turn on community plugins
→ Browse → 이미 설치된 플러그인들이 보임
→ 각 플러그인 "Enable" 버튼 클릭
```

### 3단계: 주요 플러그인 설정

#### Templater 설정
- Template folder: `templates`
- Trigger Templater on new file creation: ✅

#### Dataview 설정
- Enable JavaScript Queries: ✅
- Enable Inline Queries: ✅

#### Folder Notes 설정
- Folder Note Name: `README`
- Enable for all folders: ✅

### 4단계: 기본 설정 확인 (이미 적용됨)

#### Files & Links
- ✅ Default location: `00-inbox`
- ✅ Attachment folder: `99-attachments`
- ✅ Always update links: On

#### Editor
- ✅ Show line numbers: On
- ✅ Strict line breaks: On
- ✅ Tab size: 2

## 6. 워크플로우

### 새 프로젝트 시작
1. Cmd+N (새 노트)
2. 템플릿 선택: "프로젝트_현장_템플릿"
3. YAML 속성 입력
4. 프로젝트 폴더로 이동

### 학습 내용 기록
1. 템플릿: "학습노트_템플릿"
2. 카테고리 선택
3. `08_학습_노트/`에 저장

### 레퍼런스 추가
1. 템플릿: "레퍼런스_템플릿"
2. 출처 및 URL 기록
3. `09_레퍼런스/` 하위 폴더에 저장

## 7. 유용한 단축키

- `Cmd+P`: Quick switcher (파일 빠른 검색)
- `Cmd+O`: 파일 열기
- `Cmd+N`: 새 노트
- `Cmd+E`: 편집/미리보기 토글
- `Cmd+K`: 링크 삽입
- `Cmd+Click`: 새 탭에서 링크 열기

## 8. Dataview 쿼리 예시

### 진행중인 프로젝트
```dataview
TABLE status, date_start, primary_kpi
FROM "07_프로젝트_데이터베이스/현장별_프로젝트"
WHERE type = "프로젝트" AND status = "진행중"
SORT date_start DESC
```

### 최근 학습 노트
```dataview
TABLE category, topic, status
FROM "08_학습_노트"
WHERE type = "학습노트"
SORT date_created DESC
LIMIT 10
```

### 높은 우선순위 레퍼런스
```dataview
LIST
FROM "09_레퍼런스"
WHERE relevance = "높음" AND status = "미검토"
```

## 9. 백링크 활용

- 관련 문서 연결: `[[문서명]]`
- 섹션 링크: `[[문서명#섹션명]]`
- 별칭 사용: `[[문서명|표시할 텍스트]]`

### 예시
```markdown
이 전략은 [[03_광고_캠페인/네이버_검색광고/README#키워드 전략]]을 참고했습니다.
```
